Add an interactive Demo Mode / Guided Walkthrough system for Candy CrackZZZ.

Goal:
When someone logs in for the first time, the app should offer a guided demo mode that walks them through the features they have access to. This is mainly for the admin section, but there should also be a customer-side demo showing how ordering, rewards, referrals, review photos, and the order helper work.

Important:
Do not break the existing app.
Do not overwrite real products, merch, orders, rewards profiles, reviews, settings, or admin users.
Do not create fake live orders unless clearly marked as demo/test data.
Do not send real notifications, SMS, emails, push notifications, or customer messages from demo mode.
Demo mode must be safe and clearly labeled.

Main concept:
Create a guided onboarding system that detects the logged-in user’s role and only shows the walkthroughs for features they are allowed to use.

User roles to support:
- Builder
- Site Admin
- System Admin
- Campaign / Rewards Admin
- Staff
- Delivery Driver
- Viewer
- Customer / Rewards Member

Demo Mode entry points:
1. First login prompt:
When a user logs in for the first time, show a friendly prompt:
“Would you like a quick guided tour of your dashboard?”

Buttons:
- Start Demo
- Skip for Now
- Don’t Show Again

2. Admin dashboard button:
Add a button on the admin dashboard:
“Start Guided Demo”

3. Help menu:
Add a small Help / Demo / Tour button in the admin layout.

4. Customer side:
Add optional “How It Works” or “Try Demo” links for rewards, referral, checkout, and order helper.

Demo mode behavior:
- Use tooltips, highlighted areas, and next/back steps.
- Show one feature at a time.
- Do not overload the screen.
- Allow Skip, Back, Next, and Finish.
- Save whether each user has completed or skipped the demo.
- Let users replay the demo later.
- Make the demo mobile-friendly.
