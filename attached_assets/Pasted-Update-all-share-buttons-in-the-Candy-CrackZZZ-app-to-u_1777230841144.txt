Update all share buttons in the Candy CrackZZZ app to use the phone’s native iOS/Android share sheet first.

Goal:
Whenever a customer or admin taps a Share button, the app should open the device’s native share menu when supported. On iPhone this should look like the built-in iOS share box with options such as AirDrop, Messages, Mail, Copy Link, Quick Note, and View More. On Android it should open the Android system share sheet.

Important:
Do not manually recreate AirDrop, Messages, Mail, contacts, or app shortcuts. Those are controlled by the phone’s operating system. The app should trigger the native share sheet using the Web Share API.

Use:
navigator.share()

Apply this to all share buttons, including:
- Referral code share buttons
- Product share buttons, if any
- Merch share buttons, if any
- Menu/item share buttons, if any
- Rewards/referral share areas
- Any app/site share button that exists now or gets added during this pass

Use the most specific share info available.

For general app/share buttons:
- Title: Candy CrackZZZ
- Text: Check out Candy CrackZZZ
- URL: current app/page URL

For product share buttons:
- Title: product name
- Text: Check out this Candy CrackZZZ item: product name
- URL: current product page URL

For merch share buttons:
- Title: merch item name
- Text: Check out this Candy CrackZZZ merch: merch item name
- URL: current merch/product page URL if available, otherwise current page URL

For referral code share buttons:
- Title: Candy CrackZZZ Referral
- Text: Use my Candy CrackZZZ referral code [CODE] when you order and we can both earn rewards.
- URL: rewards page or current referral URL

Do not invent rewards, discounts, or point values. Only mention exact reward amounts if referrals are enabled and those values exist in settings.

Native share behavior:
1. On click/tap, check if navigator.share exists.
2. If supported, call navigator.share({ title, text, url }).
3. If the user cancels the native share sheet, do not show an error.
4. If native share fails because unsupported or blocked, show the fallback modal.
5. Do not show a custom popup before trying native share.
6. Do not use a fake iPhone share UI.

Fallback modal:
If Web Share API is not supported on desktop or older browsers, show a simple custom fallback modal with:
- Copy Link
- Copy Message, if useful
- Share by Text Message
- Share by Email
- Close button

Fallback links:
SMS:
sms:?&body=ENCODED_MESSAGE_AND_URL

Email:
mailto:?subject=ENCODED_TITLE&body=ENCODED_MESSAGE_AND_URL

Copy Link:
copies the URL to clipboard and shows a toast:
“Link copied”

Copy Message:
copies the full share message and URL and shows a toast:
“Share message copied”
