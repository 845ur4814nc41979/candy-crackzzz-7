Build a complete mobile-first web app called Candy Crackzzz.

Use the uploaded Candy Crackzzz logo as the starting brand asset. The app should feel fun, bold, colorful, modern, candy-shop styled, and easy to use on a phone, while still looking polished on desktop. The design language should use bright candy colors inspired by the logo: electric blue, hot pink, purple, white highlights, and dark outlines or dark contrast areas. The site should feel playful and energetic without becoming cluttered or childish.

This app is for a candy business that sells candy-coated fruit, trays, custom treats, and seasonal specials. The app needs both a public customer-facing storefront and a lightweight admin management area. Build the app so it is usable right away as an ordering/request site even before real payments are fully enabled.

Very important overall rules:
- Build this as a real app shell with working frontend flows and clean structure.
- Make it mobile-first.
- Make navigation simple and obvious.
- Large tap targets, clean spacing, clear forms.
- Do not overcomplicate the first version.
- Keep the code organized so features can be expanded later.
- Prioritize usability on iPhone-sized screens.
- Include an admin settings area where certain features can be toggled on or off.
- Payment methods must be toggleable.
- Square must exist as an optional future payment method, but must be OFF by default and hidden from customers unless enabled by the admin.
- The customer should never see disabled payment options.
- Build the app so it still works well as an order-request app even when online payments are disabled.

GENERAL APP GOAL
Customers should be able to browse candy items, view photos, read descriptions, select options, submit an order request, and optionally pay only if payments are enabled. The owner should be able to upload product photos, create and edit menu items, control business settings, and toggle payment methods on and off without changing code.
