STAGE 2 — Fix referral points transfer

Problem:
Referral points are not transferring correctly to either person when:
1. An existing rewards member shares a referral link/code.
2. A new customer clicks the referral link.
3. The new customer creates a rewards account.
4. The new customer makes a purchase.
5. The purchase is completed/confirmed.

Definitions:
- Referrer = the existing customer who shared their referral code.
- Referred customer = the new customer who used the referral code.

Expected behavior:
When a new customer signs up using a referral code and completes their first eligible order, both sides should receive the correct referral bonus points if referral rewards are enabled.

Settings to respect:
- enableRewards
- enableReferrals
- referralReferrerBonusPoints
- referralReferredCustomerBonusPoints
- referralBonusOnFirstCompletedOrder
- rewardsAwardOnCompletedOrder

Referral signup rules:
1. When a new customer joins rewards using /rewards?ref=CODE or /rewards/signup?ref=CODE, save that code on the new customer’s rewards profile as referredByCode.
2. Normalize referral codes so lowercase/uppercase/spaces do not break matching.
3. Do not allow a customer to refer themselves.
4. Do not immediately award points if referralBonusOnFirstCompletedOrder is true.
5. If referralBonusOnFirstCompletedOrder is false, award according to existing setting behavior, but do not double-award.
6. If a customer already has a rewards profile, do not create a duplicate profile for the same phone number.

Checkout/order rules:
1. If a customer already has a rewards profile with referredByCode, the order should carry enough information to process referral rewards later.
2. If a customer enters a referral code at checkout and does not already have referredByCode, save it to their rewards profile.
3. If a new rewards profile is created during checkout, save referralCodeUsed or referredByCode correctly.
4. The referral code used at checkout should be normalized before saving.
5. The order should store:
   - referralCodeUsed
   - rewardsProfileId if available
   - referralBonusStatus: pending / applied / ineligible / cancelled
   - referralBonusAppliedAt if applied
   - referralReferrerProfileId if found
   - referralReferredProfileId if found

Referral bonus awarding rules:
Award referral bonus points only when the referred customer’s first eligible order reaches the correct final status, such as completed, or whatever the app uses for completed/confirmed rewards.

When the referred customer’s first eligible order is completed:
1. Find the referred customer’s rewards profile by order phone/rewardsProfileId.
2. Find referredByCode on that profile, or referralCodeUsed on the order.
3. Find the referrer rewards profile whose referralCode matches that code.
4. Confirm referrer exists.
5. Confirm referrer is not the same customer/profile.
6. Confirm this referred customer has not already triggered a referral bonus before.
7. Add referralReferredCustomerBonusPoints to the referred customer.
8. Add referralReferrerBonusPoints to the referrer.
9. Add rewards history entries to both profiles.
10. Increment referrer successfulReferralCount.
11. Increment referrer lifetimeReferralPointsEarned.
12. Mark order referralBonusStatus as applied.
13. Set referralBonusAppliedAt.
14. Prevent double awarding if the order is saved/updated multiple times.

Rewards history entries:
For referred customer:
- type: bonus
- points: referralReferredCustomerBonusPoints
- note: “Referral signup bonus from code CODE”
- orderId if available
- createdAt

For referrer:
- type: bonus
- points: referralReferrerBonusPoints
- note: “Referral bonus for referred customer FIRSTNAME or masked phone”
- orderId if available
- createdAt

Duplicate prevention:
Do not award referral bonus again if:
- the same referred customer places a second order
- the same order is marked completed twice
- the status changes away from completed and back to completed
- the admin refreshes or saves the order again
- referralBonusStatus is already applied
- the referred customer already has a successful referral award history for that referredByCode

Admin order screen:
Show referral status clearly:
- No referral used
- Referral pending until first completed order
- Referral applied
- Referral ineligible
- Referral cancelled

Customer rewards page:
For the referred customer, show their referral bonus after it is applied.
For the referrer, show successful referral count and referral bonus history.
Do not show private details of the referred customer publicly.
